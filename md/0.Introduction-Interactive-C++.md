
# A short tutorial for how to use Cling interactive C++ interpreter

Before You run, check Cling version first to avoid inconsistent code errors.

## Summary of Metaprocessor commands
Cling understands some meta-commands that are not valid C++. Those commands are usefull to instruct cling to carry out certain administrative tasks, such as

    .x test.cpp â€“ load a file (if the file defines void test() it will be executed)
    .L libname â€“ load a libary or file(.cpp)
    .x filename.cxx â€“ loads filename and calls void filename() if defined
    .I path â€“ adds an include path
    .printAST â€“ shows the abstract syntax tree after each processed entity
    .q â€“ exit cling
    .help â€“ display a brief description

### tips 2. multi class definition could be on the same cell.


```c++
#include <iostream>
using namespace std;

class B1
{
public:
    B1(int i)
    {
        cout << "B1 " << i << endl;
    }
};

class B2
{
public:
    B2(int i)
    {
        cout << "B2 " << i << endl;
    }
};

class B3
{
public:
    B3()
    {
        cout << "B3 " << endl;
    }
};

class Derived : public B2,  public B1, virtual public B3
{
public:
    Derived(int a, int b, int c, int d, int e): B1(a), b2(d), b1(c), B2(b)
    {
        m = e;
        cout << "Derived"<<endl;
    }
protected:
    B1 b1;
    B2 b2;
    B3 b3;
    int m;
};
```




    




```c++
Derived test(1, 2, 3, 4, 5);
```
